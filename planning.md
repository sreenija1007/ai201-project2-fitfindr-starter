# FitFindr — planning.md

## Tools

### Tool 1: search_listings

**What it does:**
Scans the mock dataset to pull up clothing listings that match what the user wants to buy.

**Input parameters:**
- `description` (str): Text describing the type of clothing or aesthetic.
- `size` (str): The size the user is hunting for.
- `max_price` (float): The top dollar amount the user wants to spend.

**What it returns:**
An array of item objects that fit the criteria, with details like price, platform, and condition.

**What happens if it fails or returns nothing:**
It hands back an empty array. The agent checks for this, stops the workflow right there, and tells the user that nothing popped up so they know to tweak their search terms.

---

### Tool 2: suggest_outfit

**What it does:**
Takes the top item found from the database and uses an LLM to build a cool outfit out of it and the user's existing wardrobe.

**Input parameters:**
- `new_item` (dict): The specific item object chosen from the search step.
- `wardrobe` (dict): The object containing all items currently in the user's closet.

**What it returns:**
A text string containing personalized styling recommendations and outfit combinations.

**What happens if it fails or returns nothing:**
If the user's wardrobe file doesn't have any items in it, the tool avoids crashing by offering general fashion advice using universal basics that go with the new piece.

---

### Tool 3: create_fit_card

**What it does:**
Takes the outfit ideas and transforms them into a short, casual caption that is ready to share on social media.

**Input parameters:**
- `outfit` (str): The outfit text generated by the previous styling tool.
- `new_item` (dict): The item object used to inject things like the price or item name into the post.

**What it returns:**
A brief text string written in a casual style with emojis.

**What happens if it fails or returns nothing:**
If the styling text input is blank, it drops a simple fallback caption that just mentions the item details and jokes that the styling AI is taking a break.

## Planning Loop

**How does your agent decide which tool to call next?**
The agent bases its next steps entirely on what the search tool returns. First, it triggers `search_listings`. It evaluates the length of the returned array of items. If the length is zero, it completely cuts off the rest of the workflow, writes a helpful message to the session error path, and finishes early. If it finds matches, it grabs the first item, saves it to memory, and runs the outfit suggestion tool followed immediately by the fit card tool.

---

## State Management

**How does information from one tool get passed to the next?**
We use a single session state dictionary that lives through the whole interaction. When `search_listings` finds an item, the agent drops it into `session['selected_item']`. The outfit tool reads its inputs right out of `session['selected_item']` and `session['wardrobe']`. The text output from the outfit tool then gets saved into `session['outfit_suggestion']` so the fit card tool can easily pull it.

---

## Error Handling

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | Halts the process and returns a friendly message telling the user to try dropping the price filter or changing sizes. |
| suggest_outfit | Wardrobe is empty | Ignores the empty closet file and suggests styling the new piece with simple items like plain denim or white tees. |
| create_fit_card | Outfit input is missing or incomplete | Generates a quick caption about the thrift find itself while leaving out the missing styling text. |

## A Complete Interaction (Step by Step)

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
The agent runs `search_listings` with the description, a max price of 30, and no size parameter. It finds a $22 Faded Band Tee from Depop and saves that item object straight into the session state.

**Step 2:**
The agent sees that the search succeeded, so it passes the Faded Band Tee data along with the user's closet file into `suggest_outfit`. The tool returns a text paragraph suggesting they pair the tee with their baggy jeans and roll up the sleeves.

**Step 3:**
The agent takes that new styling text and the original tee data and passes them to `create_fit_card`. The tool creates a casual social media caption.

**Final output to user:**
The interface updates to show the user three clear outputs: the listing info for the $22 band tee, the custom paragraph explaining how to wear it, and the final shareable caption with emojis.
